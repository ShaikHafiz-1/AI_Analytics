SUPPLIER QUERY FIX PLAN
======================

PROBLEM IDENTIFIED
==================

When querying "List suppliers at location AVC11_F01C01", the system returns:
"No supplier information found for location AVC11_F01C01"

ROOT CAUSE
==========

The issue is in function_app.py, line 647:

    detail_records = ctx.get("detailRecords", [])
    suppliers = get_suppliers_for_location(detail_records, location)

The detailRecords are being passed as raw dictionaries with CSV field names:
- LOCID (not location_id)
- LOCFR (not supplier)
- GSCFSCTQTY (not forecast_qty)
- etc.

But response_builder.py expects ComparedRecord objects with attributes:
- location_id
- supplier
- qty_delta
- design_changed
- etc.

When get_suppliers_for_location() tries to access these attributes:

    location_records = [r for r in records if getattr(r, 'location_id', '').upper() == location.upper()]

It fails silently because dictionaries don't have 'location_id' attribute, so getattr returns empty string, and no records match.

VERIFICATION
============

1. Check what type of objects are in detailRecords:
   - If they're dicts with keys like "LOCID", "LOCFR" → Problem confirmed
   - If they're ComparedRecord objects → Different issue

2. Check the data flow:
   - Where does detailRecords come from in the context?
   - Is it normalized before being passed to supplier queries?

SOLUTION
========

Option 1: Normalize detailRecords in the supplier query handler
--------------------------------------------------------------

In function_app.py, _generate_supplier_by_location_answer():

    def _generate_supplier_by_location_answer(question: str, ctx: dict, scope_type: str, scope_value: str) -> str:
        if scope_type != "location":
            return "Please specify a location to analyze suppliers."
        
        location = scope_value
        detail_records = ctx.get("detailRecords", [])
        
        # NORMALIZE: Convert raw dicts to ComparedRecord objects
        detail_records = _normalize_detail_records_for_supplier_query(detail_records)
        
        # Import helper functions from response_builder
        from response_builder import get_suppliers_for_location, compute_supplier_metrics, analyze_supplier_behavior
        
        # Get suppliers for this location
        suppliers = get_suppliers_for_location(detail_records, location)
        ...

Then add a normalization function:

    def _normalize_detail_records_for_supplier_query(records):
        """Convert raw CSV dicts to objects with expected attributes."""
        from models import ComparedRecord
        
        normalized = []
        for r in records:
            if isinstance(r, dict):
                # Map CSV field names to ComparedRecord attributes
                normalized.append(ComparedRecord(
                    location_id=r.get("LOCID", ""),
                    material_group=r.get("GSCEQUIPCAT", ""),
                    material_id=r.get("PRDID", ""),
                    supplier_current=r.get("LOCFR", ""),
                    supplier_previous=r.get("LOCFR", ""),  # Assume no change if not provided
                    forecast_qty_current=r.get("GSCFSCTQTY"),
                    forecast_qty_previous=r.get("GSCPREVFCSTQTY"),
                    roj_current=r.get("GSCCONROJDATE"),
                    roj_previous=r.get("GSCPREVROJNBD"),
                    bod_current=r.get("ZCOIBODVER"),
                    bod_previous=r.get("ZCOIBODVER"),
                    ff_current=r.get("ZCOIFORMFACT"),
                    ff_previous=r.get("ZCOIFORMFACT"),
                    roc_region=r.get("ROC"),
                    dc_site=r.get("ZCOIDCID"),
                    metro=r.get("ZCOIMETROID"),
                    country=r.get("ZCOICOUNTRY"),
                    supplier_date=r.get("GSCSUPLDATE"),
                    is_supplier_date_missing=r.get("Is_SupplierDateMissing", False),
                ))
            else:
                # Already a ComparedRecord or similar object
                normalized.append(r)
        
        return normalized
