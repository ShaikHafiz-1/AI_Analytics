WHAT WAS FIXED
================================================================================

BUG CONDITION:
The Copilot UI's `/api/explain` endpoint was not routing natural language 
questions through the NLP layer, causing repeated/generic responses instead of 
properly processed, context-aware answers.

ROOT CAUSE:
The explain endpoint called ReasoningEngine directly without first processing 
questions through NLPEndpointHandler, bypassing:
- Intent classification
- Entity extraction
- Out-of-scope detection
- Conversation history tracking
- Duplicate response prevention

SOLUTION IMPLEMENTED:
Integrated the NLP layer into the explain endpoint with 5 phases:

Phase 1: Route Through NLP Layer First
- Call NLPEndpointHandler.process_question() before ReasoningEngine
- Extract queryType, scopeType, scopeValue, confidence from NLP response
- Use NLP results to guide ReasoningEngine processing

Phase 2: Handle Out-of-Scope Questions
- Detect non-planning questions (e.g., "What is your name?")
- Return clarification response instead of planning summary
- Confidence score: 1.0 for out-of-scope questions

Phase 3: Handle Clarification Needed
- Detect when clarification is needed
- Return clarification response with guidance
- Maintain conversation context

Phase 4: Check for Duplicate Questions
- Maintain conversation history
- Detect duplicate questions
- Return cached response for duplicates (no reprocessing)

Phase 5: Use NLP Results to Guide ReasoningEngine
- Override ReasoningEngine classification with NLP results if confidence > 0.8
- Pass NLP scope information for scoped metrics computation
- Include NLP metadata in response
