================================================================================
PROBLEM IDENTIFIED
================================================================================

The Copilot UI is sending natural language questions, but the backend is NOT
processing them through the NLP/LLM layer. Instead, it returns default responses.

EVIDENCE:
  ❌ Query: "List suppliers for CYS20_F01C01"
     Response: Same generic response repeated twice

  ❌ Query: "Which materials are affected?"
     Response: Ignored (no processing)

  ❌ Query: "What is your name?"
     Response: Planning summary (wrong response type)

  ❌ Query: "Is this demand-driven or design-driven?"
     Response: Ignored (no processing)

ROOT CAUSE:
  • function_app.py expects structured query_type parameter
  • Copilot sends natural language questions
  • No NLP layer to convert questions → query_type
  • Questions are ignored or return default response

================================================================================
SOLUTION CREATED
================================================================================

NEW FILE: planning_intelligence/nlp_endpoint.py

This file contains:
  ✅ NLPEndpointHandler class - Orchestrates NLP processing
  ✅ handle_nlp_query() function - HTTP endpoint handler
  ✅ Out-of-scope detection - Handles non-planning questions
  ✅ Conversation history tracking - Multi-turn support
  ✅ Intent classification - Routes to correct handler
  ✅ Entity extraction - Identifies location, supplier, material
  ✅ Response generation - Uses existing templates

KEY FEATURES:
  ✅ Natural language question processing
  ✅ Intent classification (5 types)
  ✅ Entity extraction
  ✅ Multi-turn conversation support
  ✅ Out-of-scope question handling
  ✅ Conversation history tracking
  ✅ Error handling

================================================================================
HOW TO INTEGRATE (30 MINUTES)
================================================================================

STEP 1: Add Import to function_app.py
  from nlp_endpoint import handle_nlp_query

STEP 2: Add HTTP Route to function_app.py
  @app.route(route="planning_intelligence_nlp", methods=["POST"])
  def planning_intelligence_nlp(req: func.HttpRequest) -> func.HttpResponse:
      return handle_nlp_query(req)

STEP 3: Update Copilot UI
  Change endpoint from:
    /api/planning_intelligence

    /api/planning_intelligence_nlp

  Change request format from:

  To:
    {"question": "What's the planning status?", "detail_records": [...]}

   Pattern: "What is your name?"
   Response: Clarification about capabilities

================================================================================
MULTI-TURN CONVERSATION SUPPORT
================================================================================

The solution supports multi-turn conversations:

Turn 1:
  Input: "What's the planning status?"
  Output: Summary response + conversation_history

Turn 2:
  Input: "Which location has the most changes?"
  Input: conversation_history from Turn 1
  Output: Location analysis + updated conversation_history

Turn 3:
  Input: "Are there design changes?"
  Input: conversation_history from Turn 1-2
  Output: Design analysis + updated conversation_history

Conversation history is maintained and returned with each response.
