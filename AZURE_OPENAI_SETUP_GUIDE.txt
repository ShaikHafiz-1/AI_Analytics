================================================================================
STEP 1: CREATE AZURE OPENAI RESOURCE
================================================================================

In Azure Portal:

1. Click "Create a resource"
2. Search for "Azure OpenAI Service"
3. Click "Create"
4. Fill in the form:
   - Resource Group: (create new or select existing)
   - Region: East US (or your preferred region)
   - Name: planning-intelligence-openai (or your choice)
   - Pricing Tier: Standard

5. Click "Create" and wait for deployment

6. Once deployed, go to the resource
7. Click "Keys and Endpoint" in the left menu
8. Copy:
   - Key 1 (or Key 2)
   - Endpoint

================================================================================
STEP 2: CREATE DEPLOYMENT
================================================================================

In Azure OpenAI Studio:

1. Go to Azure OpenAI Studio: https://oai.azure.com/
2. Click "Deployments" in the left menu
3. Click "Create new deployment"
4. Fill in:
   - Deployment name: gpt-4 (or gpt-3.5-turbo for cost savings)
   - Model: gpt-4 (or gpt-3.5-turbo)
   - Model version: Latest
   - Scale type: Standard
   - Tokens per minute: 40K (or your quota)

5. Click "Create"

Note: You can use gpt-3.5-turbo for cost savings:
  - GPT-4: $0.03 per 1K input tokens, $0.06 per 1K output tokens
  - GPT-3.5: $0.0005 per 1K input tokens, $0.0015 per 1K output tokens

================================================================================
STEP 3: SET ENVIRONMENT VARIABLES
================================================================================

Option A: Local Development (local.settings.json)
─────────────────────────────────────────────────

Edit planning_intelligence/local.settings.json:

{
  "IsEncrypted": false,
  "Values": {
    "AzureWebJobsStorage": "...",
    "FUNCTIONS_WORKER_RUNTIME": "python",
    "AZURE_OPENAI_KEY": "your-api-key-here",
    "AZURE_OPENAI_ENDPOINT": "https://your-resource.openai.azure.com/",
    "AZURE_OPENAI_DEPLOYMENT_NAME": "gpt-4",
    "AZURE_OPENAI_API_VERSION": "2024-02-15-preview"
  }
}

Option B: Azure Functions (Application Settings)
────────────────────────────────────────────────

In Azure Portal:

1. Go to your Function App
2. Click "Configuration" in the left menu
3. Click "New application setting"
4. Add each setting:
   - Name: AZURE_OPENAI_KEY
     Value: your-api-key
   - Name: AZURE_OPENAI_ENDPOINT
     Value: https://your-resource.openai.azure.com/
   - Name: AZURE_OPENAI_DEPLOYMENT_NAME
     Value: gpt-4
   - Name: AZURE_OPENAI_API_VERSION
     Value: 2024-02-15-preview

5. Click "Save"

Option C: Environment Variables (Windows)
──────────────────────────────────────────

In PowerShell (as Administrator):

$env:AZURE_OPENAI_KEY = "your-api-key"
$env:AZURE_OPENAI_ENDPOINT = "https://your-resource.openai.azure.com/"
$env:AZURE_OPENAI_DEPLOYMENT_NAME = "gpt-4"
$env:AZURE_OPENAI_API_VERSION = "2024-02-15-preview"

Or set permanently:

[Environment]::SetEnvironmentVariable("AZURE_OPENAI_KEY", "your-api-key", "User")
[Environment]::SetEnvironmentVariable("AZURE_OPENAI_ENDPOINT", "https://your-resource.openai.azure.com/", "User")
[Environment]::SetEnvironmentVariable("AZURE_OPENAI_DEPLOYMENT_NAME", "gpt-4", "User")
[Environment]::SetEnvironmentVariable("AZURE_OPENAI_API_VERSION", "2024-02-15-preview", "User")

================================================================================
MONITORING & TROUBLESHOOTING
================================================================================

Monitor Azure OpenAI Usage:

1. Go to Azure Portal
2. Navigate to your Azure OpenAI resource
3. Click "Monitoring" → "Metrics"
4. View:
   - Tokens used
   - API calls
   - Errors
   - Latency

Check Logs:

1. In Azure Portal, go to Function App
2. Click "Log stream" to see real-time logs
3. Look for:
   - "Azure OpenAI integration initialized" (success)
   - "Failed to initialize Azure OpenAI" (error)
   - "Using Azure OpenAI for question" (in use)

Troubleshooting:

Issue: "Missing credentials" error
  Solution: Check environment variables are set correctly
  
Issue: "Deployment not found" error
  Solution: Verify AZURE_OPENAI_DEPLOYMENT_NAME matches your deployment
  
Issue: "Invalid API key" error
  Solution: Check API key is correct and not expired
  
Issue: "Timeout" error
  Solution: Check network connectivity, increase timeout if needed
  
Issue: "Rate limit exceeded" error
  Solution: Reduce query volume or upgrade Azure OpenAI quota
